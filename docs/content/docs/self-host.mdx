---
title: 自托管引导节点
description: 自建引导节点，连发现层都不依赖第三方。
---

SwarmDrop 自带一个公共引导节点，开箱即用。如果你想**连发现层都自己掌控**，可以自建。

## 为什么要自建

- **可用性**：不依赖某个公共节点的存活。
- **隐私**：DHT 的发现流量也只经过你的基础设施。
- **内网部署**：在企业 / 家庭网络里跑一个长期在线的入口。

引导节点只参与 **DHT 路由**，它**看不到任何传输内容**，作用相当于一本「电话簿入口」。

## 运行一个节点

引导节点是一个长期在线、有公网地址的 libp2p 节点，同时监听 TCP 与 QUIC：

```bash
# 监听端口（示例 4001），需在防火墙 / 安全组放行 TCP + UDP
swarmdrop-bootstrap \
  --listen /ip4/0.0.0.0/tcp/4001 \
  --listen /ip4/0.0.0.0/udp/4001/quic-v1
```

> 公共引导节点示例：`47.115.172.218:4001`（TCP + QUIC）。自建时换成你自己的地址。

## 让客户端连上它

在应用的网络设置里，把你的引导节点 multiaddr 加进 bootstrap 列表：

```
/ip4/你的公网IP/tcp/4001/p2p/<PeerId>
/ip4/你的公网IP/udp/4001/quic-v1/p2p/<PeerId>
```

`PeerId` 在引导节点启动时打印。客户端启动后会自动向它做 DHT bootstrap，加入网络。

## 注意事项

- **稳定的公网地址**：引导节点需要可被外部访问，建议放在有公网 IP 的服务器上。
- **端口放行**：TCP 与 UDP（QUIC）都要开。
- **不是中继**：引导节点只做发现。NAT 穿透由 DCUtR 与 Relay 负责，是另一组角色。
