SwarmDrop 的安全目标很直接：**除了收发双方，没有任何一方能读到文件内容**，包括引导节点和中继节点。

## 密钥体系

| 用途 | 算法 | 存放位置 |
| --- | --- | --- |
| 设备身份 | Ed25519 密钥对 | Stronghold 加密保险库 |
| 传输会话 | 每次独立的 256-bit 对称密钥 | 仅内存，传输结束即弃 |
| 完整性校验 | BLAKE3 | 随分块传输 |

设备私钥**永不离开本机**。每次传输都会协商一把全新的会话密钥，即便某次密钥泄露，也无法回溯解密历史传输。

## 端到端加密

文件在发送端分块后即用 **XChaCha20-Poly1305** 加密，到接收端才解密。

中继节点转发的是密文，它没有会话密钥，**无法解密**。打洞成功时甚至不存在中继。

## 零信任

- **引导节点**：只帮新节点加入 DHT，看不到传输内容。
- **中继节点**：只转发密文，没有解密能力。
- **无遥测**：不收集任何用户数据，不连分析后台。

## 本地保护

- **生物识别解锁**：TouchID / FaceID / Windows Hello，本地校验后才解锁 Stronghold。
- **密码保护保险库**：私钥由你的密码加密，应用本身也读不到明文私钥。

## 威胁边界

SwarmDrop 保护**传输内容**与**设备私钥**。它不负责：

- 接收端落盘后的文件安全（由你的操作系统与磁盘加密负责）。
- 对方设备本身是否可信（配对意味着你信任那台设备）。
