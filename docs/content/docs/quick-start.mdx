只要两台设备各装一个 SwarmDrop，两分钟就能开始传文件。

## 下载安装

到 [Releases](https://github.com/swarm-apps/SwarmDrop/releases/latest) 选你的平台：

| 平台 | 格式 |
| --- | --- |
| **Windows** | `.msi` · `.exe`（x64） |
| **macOS** | `.dmg`（Apple Silicon · Intel） |
| **Linux** | `.deb` · `.rpm` · `.AppImage`（x64） |
| **Android** | `.apk`（arm64-v8a）—— 见 [Releases](https://github.com/swarm-apps/SwarmDrop/releases) 里的 `mobile-v*` |

## 第一次传输

<Steps>

<Step>

### 启动节点

打开应用，设置一个安全密码并启动 P2P 节点。
密码用来保护设备私钥，私钥落 [Stronghold](https://docs.rs/iota-stronghold) 加密保险库，永不离开本机。

</Step>

<Step>

### 配对设备

有两种配对方式，选一种：

- **配对码**（跨网络）：一方生成 6 位数字码，对方输入即可。
- **局域网**（同一 Wi-Fi）：自动发现附近设备，点击配对。

配对一次，之后互为信任设备，无需重复。

</Step>

<Step>

### 拖拽发送

选中已配对的设备，把文件或文件夹拖进窗口。
SwarmDrop 自动选最快路径、加密直传，并显示实时进度。完成后双方收到通知。

</Step>

</Steps>

## 传输路径（自动选优）

SwarmDrop 不需要你关心网络细节，它会按这个顺序尝试：

| 路径 | 延迟 | 触发条件 |
| --- | --- | --- |
| 局域网直连 | ~2 ms | 同一网络 |
| NAT 打洞 (DCUtR) | 10-100 ms | 不同网络，打洞成功 |
| 中继转发 | 100-500 ms | 打洞失败兜底 |

## 从源码构建

需要 Node 18+、pnpm 11+、Rust 1.80+。

```bash
git clone --recurse-submodules git@github.com:swarm-apps/SwarmDrop.git
cd SwarmDrop
pnpm install

pnpm tauri dev      # 开发
pnpm tauri build    # 打包
```

接下来可以读一读[核心概念](/docs/concepts)，理解配对码和 NAT 穿透背后发生了什么。
