# Product

## Register

product

## Users

- **隐私敏感用户**：不想让文件经过任何第三方服务器/云盘中转。
- **跨生态用户**：同时使用 macOS / Windows / Linux / Android / iOS，需要设备间无缝互传。
- **技术爱好者 / 开发者极客**：认可开源、愿意自托管引导节点，欣赏"够用就好、不故弄玄虚"的工具。
- **构建 AI agent 工作流的开发者**：把 SwarmDrop 当作 agent 产出文件后"送到另一台设备"的基础设施来用。

使用场景：设备配对完成后，用户几乎不用再想"怎么传"——选中文件、点设备，直接送达；接收是隐式的后台行为，而不是要在"发送/接收"之间做选择的模式。核心任务永远是：快速地、可信地，把文件从一台设备送到另一台设备，不管是局域网直连还是跨网络打洞/中继。

## Product Purpose

SwarmDrop 是设备与设备之间的数据通道——为人类和 AI agent 共用。它存在的原因，是消灭"传文件必须经过云盘或即时通讯软件中转"这件事带来的摩擦和隐私折损：无账号、无服务器、端到端加密，局域网直连或跨网络打洞/中继自动择优。

成功的样子：用户配对完设备后几乎感觉不到"传输"这件事本身的存在——选文件、选设备，剩下的交给协议；全程加密、可断点续传、状态透明。同时，本地 MCP server 让 AI agent 能通过同一套通道发送文件、检索收件箱，把"设备间传输"变成人和 agent 共享的基础设施，而不是一个额外贴上去的"AI 功能"。

## Brand Personality

**3-5 词人格**：去中心化、极客、克制、可信赖、直接高效

**Voice & Tone**：技术直白，不过度营销化；用词精准，不夸大宣传。

**确认的视觉参照**（每个只取其中一个特质，不整体照搬）：
- **Raycast / Warp** — 开发者工具的极客感：精致的暗色调、等宽字体式的细节、命令行般的高效直接。
- **Arc / Linear** — 克制精准：大量留白、动效细腻但从不多余、信息层级清晰不喧哗。
- **1Password / Cryptomator** — 安全工具的可信赖感：稳重色调、把加密状态/连接类型清楚地可视化出来，而不是故弄玄虚地"看起来安全"。

## Anti-references

- **通用 SaaS 后台套路**：渐变色 hero 区、无脑铺满的卡片网格、千篇一律的仪表盘视觉。
- **厚重企业级后台**：密集表格、灰上加灰的既视感、Windows 老式管理面板风格。

## Design Principles

1. **隐式优先于选择** — 配对完成后，接收文件是后台默认行为；不需要用户在"发送 / 接收"之间做模式选择。
2. **状态诚实可见** — 加密方式、连接类型（直连/打洞/中继）、传输进度是真实状态，要像安全工具一样清楚可视化，而不是靠装饰效果掩盖或美化。
3. **极客的高效直接** — 交互路径要短、反馈要快；不为了"看起来友好"而堆砌引导层级或多余确认步骤。
4. **克制的层级** — 用留白和排版层级传达信息优先级，而不是颜色堆砌或卡片套卡片。
5. **AI 是基础设施，不是噱头** — MCP / agent 相关能力要像"另一个正常客户端"一样自然融入产品，而不是额外贴一个"AI 功能"徽章。

## Accessibility & Inclusion

标准 **WCAG AA**：文本对比度、键盘可达性等基本要求，目前没有额外的特殊人群需求。

项目里已经在为降级路径投入（`prefers-reduced-transparency` 下 glass 效果退化为纯色卡片背景），后续动效 / 视觉效果工作应延续这个"始终提供降级路径"的思路，对 `prefers-reduced-motion` 做同等对待。
