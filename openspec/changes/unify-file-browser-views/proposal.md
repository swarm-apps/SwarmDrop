## Why

SwarmDrop 当前用 `FileTree` 展示发送选择、接收确认和传输状态，却在收件箱内另写了一套 `FileCard` 网格，导致文件图标、操作、状态、滚动和视觉规则逐渐分叉。用户也无法在发送文件时切换到更适合视觉确认的卡片视图，因此需要把两套实现收敛为统一的文件浏览组件。

## What Changes

- 新增统一的 `FileBrowser` 组件，以同一份文件数据、状态和操作能力驱动树形与网格两种视图。
- 精修树形视图：文件夹展开不再使用常驻高亮背景，仅通过展开箭头、文件夹图标和层级引导线表达状态；hover、焦点和真实选中态各自有明确反馈。
- 新增树形 / 网格视图切换，并允许不同使用场景配置默认视图、可用视图和偏好作用域。
- 将现有收件箱 `FileCard` 抽取为统一网格视图；网格支持文件类型图标、可用时的本地缩略图、相对路径、大小、缺失、传输状态与上下文操作。
- 为树形和网格视图保留大文件集虚拟化与独立滚动，切换视图不得改变文件集合、排序、状态或可执行操作。
- 通过适配器统一发送选择、快捷发送、接收 Offer、传输详情 / 发送进度和收件箱详情的数据模型。
- 迁移所有现有调用点，移除收件箱页面私有的 `FileCard` 实现，并逐步替代 `FileTree` 的直接对外使用。
- 收敛并取代未完成的 `file-tree-component` 变更中已过时的 UI 契约，特别是“文件夹展开时使用 accent 背景”的要求。

## Capabilities

### New Capabilities

- `file-browser-ui`: 统一文件浏览器的数据契约、树形 / 网格视图、上下文操作、视图偏好、虚拟化、缩略图降级和可访问性交互。

### Modified Capabilities

无。当前主规格目录中尚无已归档的文件浏览 UI capability；本变更以新的 `file-browser-ui` capability 收敛旧的未完成变更。

## Impact

- 前端组件：新增 `src/components/file-browser/` 统一文件浏览模块，并移除旧的 `src/components/file-tree/` 实现。
- 页面与流程：`/send`、`/send/share-target`、收件箱详情、传输详情 / 发送进度、接收 Offer 弹窗。
- 状态与偏好：需要一个按使用场景区分的视图偏好入口，但不改变文件选择、传输或收件箱业务状态。
- 性能：树形继续复用 `@headless-tree/react` 与 `@tanstack/react-virtual`；网格需要按行虚拟化，避免大文件集合一次性渲染。
- 安全：缩略图只使用调用方显式提供且允许访问的 URL；任意发送路径默认降级为文件类型图标，不扩大 Tauri asset protocol 文件系统 scope。
- 后端、传输协议和数据库模型不变，不新增依赖。
